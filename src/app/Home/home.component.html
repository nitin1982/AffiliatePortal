Highlights

  • AppRouting, Affiliate, Security, Admin are diffenrent `Feature Modules`
  • Routing is configured in AppRouting Module
  • No security(No Login needed) is applied to Home component.
  • Affiliate Module has parent-child components.
  • Routing Guard - `CanActiavte` is applied to Affiliate Module.
  • Resolver Guard(Prefetching data to populate data in component) is applied to Performance component.
  • Admin Module is Lazy Loaded module.
  • Should use `CanLoad` RoutingGuard with lazy loaded module(Same as CanActiavte)
  • Admin module is also using pre-loading strategy.
